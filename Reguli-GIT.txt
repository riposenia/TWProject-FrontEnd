1. Fiecare branch va avea urmatoare denumire: Primele doua litere din nume+primele 2 litere din prenume + "/" + nume_sugestiv_branch 
Ex: CiAv/BugFixing   sau  CrGo/StackNavigatorCorrection  sau IaCe/IconScaling   sau  RaBo/DatabaseUpdate




2. Un branch este de unica folosinta, dupa efectuarea merge-ului unui branch, acesta va fi sters in siguranta si pentru viitoarele modificari vom da "checkout" la noul master 
din care vom creea un nou branch pentru efectuare modificarilor dorite



3. Commiturile trebuie sa fie scurte si dese si cel mai important sa aiba o descriere adecvata 



4. Pentru fiecare pull request catre master se va efectua un code review de catre ceilalti membrii ai echipei si se va decide daca acell pull trece de verificari si ajunge
sa se faca un merge pentru el.
